Updating

GM will receive free updates in the future. One of the main goals was to make sure the update progress is going to be as easy as possible. Due to its nature (template), updating can become an unpleasant task.

Therefore we’ve tried to make it as easy as possible to let you apply your own overrides (in case you would like to) to both Sass (_css/) or JavaScript (_js/) files without altering the original files. Please check Sass and JavaScript section for more info!

In general, the following core folders/files will have to be updated with each update:

Project Structure

  • Assets
    • Images/*
    • Libraries/*
    • Scripts
      • Default.js
    • Styles
      • Default.css
  • UserControls
    • Header.ascx
    • Footer.ascx
  • Default.aspx

Remember that it’s always a good idea to backup your work before updating to a new version!

JavaScript

GM custom JS functionality was built from scratch with ES6 (ECMAScript 2015). This enabled us to take advantage of many new features (ex. the let and const keyword) which weren't available in ES5 (the version most web browsers support and run) and write clean and more maintainable code.

ES6 JS code can be found under Assets/Scripts/ folder.

Files Description
Default.js

This is the main JS file containing the functionality used in the project.

Please make sure that you don’t use snippets from ES6 code directly in your pages since older browsers won't be able to run them.

    e.preventDefault();

    let elementsRequired = document.forms["form1"].querySelectorAll("[required]");
    let isOk = true;

    //check requirements elements
    for (let i = 0; i < elementsRequired.length; i++) {
        const element = elementsRequired[i];
        if (element.value == "") {
            isOk = false;
            // here is called sweetalert with error message

             element.classList.add("invalid");
        }
        else {
            element.classList.remove("invalid");
        }
    }

    //form is valid
    if (isOk) {
        // here is called sweetalert with success message

           ...

    }
getChkState

Function to chechk in localStorage all the keys name start with check_. For keys that have value true, set attribute checked on checkbox elements that have same id like localstorage key name. 

function getChkState() {
    for (let key in window.localStorage)
        if (key.indexOf('check_') == 0) {
            document.getElementById(key).checked = window.localStorage.getItem(key) == 'true';
        }
}
setChkState

Function to set a key in localstorage for every group that is collapsed or expanded to save the state. 

function setChkState(id) {
    let checkbox = document.getElementById(id)
    window.localStorage.setItem(id, checkbox.checked);
}
checkNumber

Function that is called onkeyup (when the user releases a key) to check if input[type="text"] with name number has value Test.

If condition is met, background color of input with name testField8 is changed to rgb(169,169,169).

function checkNumber() {
    if (form["number"].value == 'Test') {
        form["testField8"].style.backgroundColor = 'rgba(169,169,169)';
    }
    else {
        //reset initial background
        form["testField8"].style.backgroundColor = 'rgba(0, 0, 0, 0.1)';
    }
}
downloadImage

Asynchronous function to download image when button with id downloadImage is clicked. 

const downloadBtn = document.getElementById('downloadImage'); //get button id and put it in a constant
const imageUrl = document.getElementsByClassName('img2Down')[0].getElementsByTagName('img')[0].currentSrc; //get image source and put it in a constant

// event listener for button. when it is clicked, call function downloaImage with image url passed as argument
downloadBtn.addEventListener('click', (event) => {
    event.preventDefault();
    downloadImage(imageUrl);
})

// async function to download image
async function downloadImage(imageUrl) {
    fetch(imageUrl, {
        mode: 'no-cors',
    })
        .then(response => response.blob())
        .then(blob => {
            let blobUrl = window.URL.createObjectURL(blob);
            let a = document.createElement('a');
            a.download = imageUrl.replace(/^.*[\\\/]/, '');
            a.href = blobUrl;
            document.body.appendChild(a);
            a.click();
            a.remove();
        })
}

Quick Tips

High resolution screens

The UI will look crispy clear on high resolution screens and if you would like your images to look great, too, then you will have to use the srcset attribute of the <img> element. For example, if you have an image url/to/image.jpg and its high resolution version url/to/image@2x.jpg, you will have to use the following markup: 

<img src="url/to/image.jpg" srcset="url/to/image@2x.jpg 2x" alt="Image Description">

The browser will auto select the best version and load it accordingly. Older browsers will ignore the srcset attribute and will load the regular image as usual.

Demo Forms

Please remember to remove onsubmit="return false;" from <form> tags if you are using a form from the example pages. It is just added to prevent forms from submitting in case you click on a submit button while previewing the pages.
